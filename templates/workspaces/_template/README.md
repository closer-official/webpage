# テンプレ作業用フォルダ（雛形）

## このフォルダをコピーしたら最初にすること

1. フォルダ名を **`templates/workspaces/<スラッグ>/`** にリネーム（英小文字・ハイフン推奨）。
2. **`SPEC.md`** の `{{...}}` をすべて埋める。
3. Cursor で **このフォルダをルートとして開く**（またはワークスペースに追加）。

## いまの見え方をどこで確認するか

- **すでにギャラリーに載っている ID になったあと**は、運営向け **`/admin/template-hub.html`** で一覧・縮小プレビュー・全画面リンクがまとまっています（本番は Basic 認証下の `/admin/`）。
- **まだカタログに無い作業中**は、このフォルダの **`mock/`** を直接開くか、§4a で仮登録してから `/api/template-preview/<id>` を見ます。

## ギャラリー非表示のまま検証 → 公開

新規ビルトイン ID を **`server/data/json/galleryDraftBuiltins.json`** の `draftBuiltinIds` に入れると、**一般向けギャラリー・ヒアリングには出ません**（運営は **`/admin/template-hub.html`** でログイン後にプレビュー・**ギャラリーに公開**ボタン）。  
Supabase 運用時は同キーが DB の `app_store` に保存されます（API で公開すると下書き一覧から外れます）。

## ディレクトリの意味

| パス | 用途 |
|------|------|
| `SPEC.md` | テンプレ ID 案・表示名・向き業種・本体取り込みチェックのメモ |
| `mock/` | 静的 HTML/CSS での見た目プロトタイプ（本体への移植前） |
| `notes/` | コピー案・セクション案・スクショメモなど自由記述 |

## 本体への取り込み

`docs/テンプレ制作ワークスペース要件書.md` の **§4** を実行する。  
完了後、この `mock/` の内容は `buildHtml` 側に反映済みなら、アーカイブや削除してよい。
