# テンプレ制作ワークスペース（リポジトリ内）

**本体（運営 SPA・API・既存 LP パイプライン）** と **テンプレ増産用の作業場** を、**同じ Git・同じ Vercel** のまま **ディレクトリで分ける**ための場所です。

**Git / Vercel を初めて触る人**は、先に **`docs/テンプレ制作ワークスペース要件書.md` の「背景知識ゼロ向け」** と **`docs/開発から納品までの流れ.md`** を読んでください。

## 使い方（最短）

1. `templates/workspaces/_template/` をコピーし、  
   `templates/workspaces/<テンプレのスラッグ>/` を作る（例: `salon-neon-lp`）。
2. その **`<スラッグ>/` フォルダだけ**を Cursor で「フォルダーを開く」（推奨）。
3. `SPEC.md` を埋め、`mock/` でプロトタイプを書く。
4. 仕様が固まったら、**本体取り込み**は `docs/テンプレ制作ワークスペース要件書.md` の **§4 チェックリスト** に従い、通常は **`master` 相当のブランチで1本の PR** にまとめる。

## やらないこと

- この配下だけで完結させようとして **`server/buildHtml.js` を無計画に長期並行編集**しない（競合の温床）。  
  取り込みは **まとめて1回**か、**小さな `server/templates/...` モジュールを先に追加**してから本体に接続する。

## ドキュメント

- **コピペ用の詳細要件:** `docs/テンプレ制作ワークスペース要件書.md`
- **テンプレファイルの正の一覧:** `docs/テンプレまわりの置き場とデプロイ.md`
