# ウェブページ作成ツール

**フルオート**：Webサイトがない店を探す → 口コミから雰囲気・客層・コンセプト・強みをAIで分析 → 既定フォーマットでLP案を3つ生成（コンセプト別テンプレート）→ DM文面を自動作成 → **あなたは最終チェック（承認）だけ**。  
**手動フロー**：キューに手動/ブラウザMapsで追加 → 調査フォームで入力 → LP生成 → 検閲。PDF・テキストから直接作成も可能。

## 機能

### フルオート（サーバー起動時）
- **サーバーで収集** … 検索ワード（例: 港区 カフェ）で Places API を実行し、**Webサイト未登録の店**だけキューに投入。
- **キューを1件処理** … 口コミを取得 → Gemini で雰囲気・客層・コンセプト・強みを分析・文字起こし → コンセプトに合うテンプレート上位3つでLPを生成 → DM文面を生成。あなたは検閲ダッシュボードで確認して OK/NG のみ。

### キュー（手動／ブラウザ）
- **Google Maps で取得** … ブラウザでエリア・キーワード検索し、サイト未登録の店をキューに追加。
- **手動で追加** … Instagram などで見つけた店舗を手動でキューに追加。

### 生成オプション（スイッチでオンオフ）
- 多言語対応 / 問い合わせフォーム設置 / Instagram・LINE リンク埋め込み / 「Presented by」表記 / QRコード掲載。設定タブで一括切り替え。

### 調査 → LP生成（手動時・AIなし）
- キューから「調査」で、コンセプト・強み・イメージカラーを入力し、テンプレートでLP生成。

### 検閲ダッシュボード
- **左** … 店舗情報・実在確認用シグナル・Mapsリンク。
- **中央** … LPプレビュー（フルオート時は案1〜3を切り替え可能）。
- **右** … DM文面（編集可）。OK / NG で承認・ボツ。

### 実在しない店舗リスク対策
- 収集時に**「レビュー3件以上」などでフィルタ**可能（推奨）。
- キュー・調査・ダッシュボードで**「要確認」バッジ**（レビュー少なめ）を表示。
- 各所に**「Google Maps で確認」リンク**を表示し、送信前に実在確認を促す表示。

### PDF・テキストから直接作成
- PDF取り込み / テキスト貼り付け / 空のページから作成
- 業界・デザイン選択、編集、SEO自動設定、プレビュー、HTMLエクスポート

## 使い方

```bash
npm install
npm run dev
```

ブラウザで http://localhost:5173 を開き、ステップに従って操作します。

1. **ステップ1** … PDFを選択するか、テキストを貼り付けるか、空のページから作成
2. **ステップ2** … 業界とデザインスタイルを選択し「編集・プレビューへ進む」
3. **ステップ3** … 内容とSEOを編集し、プレビューを確認して「HTMLをダウンロード」で保存

## フルオートの起動方法

1. **サーバー**  
   ```bash
   cd server && cp .env.example .env
   # .env に GOOGLE_MAPS_API_KEY と GEMINI_API_KEY を設定
   npm run dev
   ```
2. **フロント**  
   プロジェクトルートの `.env` に `VITE_API_URL=http://localhost:3001` を追加し、`npm run dev`。キューで「サーバーで収集」「キューを1件処理」が使えます。
3. **設定タブ**で多言語・問い合わせフォーム・SNS・Presented by・QRコードのオンオフを設定。

## ドメインについて

**ドメインの取得・DNS・ホスティングへの紐づけは本アプリの外で行います。** 本ツールはHTMLの生成と承認フローまで。公開用ドメインはご自身のサーバーや Netlify / Vercel 等で設定してください。

## 技術スタック

- Vite + React (TypeScript)
- PDF.js（pdfjs-dist）でPDFテキスト抽出
- Google Maps JavaScript API（Places）／ Places API（サーバー側・フルオート用）
- サーバー: Express + Gemini API（口コミ分析・DM生成）
- 全処理はブラウザ内で完結（サーバーなしでも手動フローは利用可）

## Google Maps API キー

キューで「Google Maps で取得」を使う場合、プロジェクトルートに `.env` を作成し、次を設定してください。

```
VITE_GOOGLE_MAPS_API_KEY=あなたのAPIキー
```

Google Cloud で **Maps JavaScript API** と **Places API** を有効にしたキーが必要です。`.env.example` をコピーして `.env` にリネームして編集しても構いません。
