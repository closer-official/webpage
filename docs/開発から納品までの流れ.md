# 開発から納品までの流れ（標準: `master` ＋ `npm run deploy`）

**納品の思想（テンプレのみコードで持つ・店舗個別は触らない・購入者ページで差し替え）**は **`docs/テンプレ中心納品運用.md`** を参照。

**標準運用**: **`master` で開発**し、**`npm run deploy`**（= コミット＋現在ブランチの `git push`）で GitHub に載せる。Vercel の **Production Branch を `master` に設定**しておけば、その push で **本番デプロイも走ります**。

> **任意の別運用**: `production` ブランチを挟み、`npm run deploy:prod` で本番に出す手順は **`docs/本番ブランチ運用.md`**（参考・旧手順）。

---

## まず全体像（3つのレイヤー）

| レイヤー | 何をするか | ブランチ / 場所 |
|----------|------------|-----------------|
| **① コード開発** | テンプレ追加・ツール改修・バグ修正 | 基本 **`master`** |
| **② 本番デプロイ** | ①を顧客向けサーバーに反映 | **`npm run deploy`** で `master` を push（Vercel が本番をビルド） |
| **③ 店舗ごとの中身** | 文言・画像・キャンペーン・店舗アカウント | **Supabase（DB）**＋DNS。コードデプロイとは別作業 |

「テンプレを増やす」は **①②**、「新店舗の納品」は多くの場合 **③（毎回）** で、**②はコードを変えたときだけ**で足ります。

---

## ① 日常の開発

1. **`master` にいることを確認**  
   `git checkout master && git pull origin master`

2. **作業**  
   - 例: `public/deliverables/`、`src/lib/templates.ts`、サーバー API など

3. **リモートへ載せる（本番も同じコミットが欲しいとき）**  
   ```bash
   npm run deploy
   ```  
   - 変更をコミットして **`master` を `origin` に push**  
   - Vercel の Production Branch が **`master`** なら、**このタイミングで本番も更新**されます。

4. **動作確認**  
   - ローカル: `npm run dev` / `npm run server`  
   - 本番に出す前に型・ビルドだけでも: `npm run build`

**末尾の `vercel --prod` はデフォルトオフ**（誤って CLI 直載せしないため）。Git 連携で十分ならそのままで問題ありません。

---

## ② 本番にコードを載せる（標準）

追加のコマンドは **不要**です。**`master` で `npm run deploy`** まで終われば、Vercel 設定が合っていれば本番に反映されます。

**Vercel 側の確認（一度でよい）**

- プロジェクト → **Settings** → **Environments** → **Production** → **Branch Tracking** = **`master`**（実際に push しているブランチ名と一致させる）

---

## ③ 納品する「商品」を作る（店舗ごと）

**コードではなく「その店の LP とアカウント」** を用意します。**コードを変えていなければ `npm run deploy` は不要**です。

### 3-1. 決めること

- **サイトキー** `siteKey`（英小文字・数字・ハイフン、2〜64文字）  
  例: `shibuya-gym`  
  → 本番URL例: `https://shibuya-gym.store-official.net`

### 3-2. DNS

- `店舗キー.store-official.net` を **いつもの本番プロジェクト** に向ける

### 3-3. 店舗用 CMS アカウントの作成（運営）

- `POST /api/admin/lp-cms-provision` など（詳細は `docs/管理者画面CMS.md`）

### 3-4. 店舗へ渡すもの

- 公開URL・管理URL・ログイン情報

---

## 流れを例でつなぐ

### パターンA: テンプレやサーバーを変えて全店舗に反映

1. `master` で実装 → **`npm run deploy`**（push まで）  
2. Vercel の **Deployments** で成功を確認

### パターンB: コードは触らず、新店舗だけ納品

1. DNS ＋ `lp-cms-provision` ＋ 店舗へ連絡  
2. **`npm run deploy` は不要**

### パターンC: バグ修正を急いで本番へ

1. `master` で修正 → **`npm run deploy`** だけ

## コマンド早見表

| やりたいこと | コマンド / 操作 |
|--------------|-----------------|
| 開発をリモート＋（設定どおり）本番に載せる | **`npm run deploy`**（`master` push） |
| `production` ブランチ経由で本番（任意・旧運用） | `npm run deploy:prod` → `docs/本番ブランチ運用.md` |
| CLI で本番直載せ（非推奨・特別なときだけ） | `DEPLOY_USE_VERCEL_CLI=1 npm run deploy` |
| 新店舗の口座作成 | `POST /api/admin/lp-cms-provision` |
