# 制作運用方針（テンプレ化の流れ）

**方針（2025年3月〜）**

1. **専用を一つずつ作る**  
   案件ごとに、その人向けのLP・ページを仕上げる（ヒアリング・ダッシュボードの編集・プレビュー確認など、既存フローを利用）。

2. **完成した納品物をテンプレに保存する**  
   納品が確定したら、そのデザイン・構成を **テンプレート候補**として残す。  
   - アプリ内の **「デザイン確認」タブは廃止**済み。テンプレ候補への反映は **`POST /api/template-customizations/save`**（`src/lib/api.ts` の `saveTemplateCustomization` と同じ）で行うか、**リポジトリ内のテンプレ用JSON／プリセット**を直接編集してデプロイする運用とする。  
   - URL からの自動抽出は **叩き台用**。最終判断は人の目視・スクショ＋シートでもよい（[README](../README.md) の運用と併用可）。

3. **以降の案件**  
   蓄積したテンプレから近いものを選び、専用調整してまた納品 → 必要ならまたテンプレに追加、の繰り返し。

この流れを「先にテンプレを量産する」より **納品物からテンプレを育てる** 前提とする。

## 専用LPの置き場所（例）

お客様ごとに `public/deliverables/<案件スラッグ>/` を作り、`index.html` を納品物とする。  
（例: [日本史徹底攻略 / 東](../public/deliverables/japanese-history-higashi/)）
