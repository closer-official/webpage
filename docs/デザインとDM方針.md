# デザインとDM方針

## DM（私から店に送る文面）について

**意図:** DMは「あなた（作成者）から、そのお店に送る」挨拶文です。

- **趣旨:** 作成者はその店に**行ったことがない**前提。「口コミや評判で評価が高く、こんなに素晴らしいお店がウェブを持っていないのはもったいないと感じたから、勝手にLPを作りました」という流れで、訪れたと誤解される表現（「先日貴店を訪れ」等）は使いません。
- 店ごとに**内容をまったく変えた**文章になるよう、AIで生成しています。
- コンセプト・強みを具体的に盛り込み、「ページではその魅力をこう表現した」と伝えます。

生成後もダッシュボードでDM文面を編集できます。

## ウェブページのデザインについて

### 二つのアプローチ

| アプローチ | 内容 | メリット | デメリット |
|------------|------|----------|------------|
| **A. AIがデザイナーとして完璧にする** | 毎回、AIがレイアウト・色・ typography まで考えてページを生成する | 理論上は店ごとに唯一無二のデザインにできる | 品質のばらつきが大きい。HTML/CSSを毎回生成するため保守しづらく、実装コストも高い |
| **B. 業種ごとに既存サイトを参照し、良い所を抽出してその店用に仕上げる** | パン屋・整骨院・美容室など業種ごとに「すでにウェブがある店」の良い点を分析し、テンプレートやルールに落とし込む。その店の文言・写真・色だけ差し替える | 業界に合った見た目になりやすく、再現性が高い。テンプレートを増やせばデザインの幅を広げられる | 業種ごとの参照収集・テンプレート整備の手間がかかる |

### まずやるべきこと：既存サイトの学習から

**結論：はい。すでにあるウェブサイトの学習を先に行い、その結果でデザイン・文言のテンプレを作る順がよいです。** コストも一括で抑えられます。

1. **各業種で検索**（例: 東京 パン屋、東京 美容室、東京 整骨院 …）し、**上位表示される「ウェブあり」の店を業種あたり 60〜100 件**収集する。
2. **メタ取得（title・description・デザイン）** で各サイトの文言とデザイン特徴を取得する。
3. **「上位表示要因を言語化」** で、文言の共通項と**デザインの共通項**を業種別にまとめる。
4. その**言語化結果をもとに**、業種ごとのテンプレ（色・フォント・レイアウト・見出しのルール）を決め・実装する。
5. その後、LP生成ではそのテンプレ＋店ごとの文言で組み立てる。

**コストの目安（業種8種 × 約60件 ≒ 480件を一括で学習する場合）**

| 項目 | 内容 | 概算 |
|------|------|------|
| **Maps API** | 検索＋Place Details（60件×8業種で約480件分の詳細取得） | 約 $8〜15 程度（一括1回） |
| **Gemini（デザイン抽出）** | 1サイトあたりHTML断片を送って特徴抽出 | 約 $0.5〜1 程度（480回） |
| **Gemini（言語化）** | 業種別に共通項をまとめる分析 | 約 $0.1 未満 |
| **合計** | 初回の「学習用データ収集＋分析」 | **約 $10〜20（1,500〜3,000円程度）** |

学習は**一度やれば**、同じデータで何度でも分析し直したり、テンプレを微調整したりできます。運用中のLP生成では、学習時ほどの大量呼び出しは発生しないため、**「先に学習」のコストは高すぎず、むしろ先行投資として妥当**です。取得件数は「60件」でページネーション、「100件」にする場合は同一業種でクエリを変えて2回に分けるなどで対応できます。

**手順（アプリ上での操作）**

**方法A: 学習タブで自動実行（推奨）**

1. **学習** タブを開く。
2. 業種（1業種 or 「全業種」）と件数（20/60/100）を選び、「学習を開始」を押す。
3. **収集 → メタ・デザイン取得 → 分析** が自動で実行される。完了まで待つ（業種数・件数により数分〜数十分）。
4. 完了後、同じ画面に「学習結果」が表示される。**テンプレ作成の参考**（文言の傾向・デザインの共通項・業種別）を確認し、必要に応じて `server/conceptTemplates.js` を編集する。

**方法B: 手動で参照サイトから（従来どおり）**

1. キュー画面の「参照サイト（ウェブあり・上位表示分析）」を開く。
2. 取得件数を **60件** または **100件** にし、業種ごとに検索（例: `東京 パン屋`）→「ウェブありで収集」を実行。
3. 「メタ取得（title・description・デザイン）」→「上位表示要因を言語化」を実行。
4. 表示された結果を元に、`server/conceptTemplates.js` を調整する。

### 推奨

**「Bを土台にしつつ、必要に応じてAの要素を足す」ハイブリッド**が確実です。

1. **まずB（業種参照）**
   - 各業種（パン屋・整骨院・美容室・カフェ・小売・サービスなど）で、すでにウェブがある店の良いLPを数サイトずつ参照する。
   - 共通する「良いところ」（余白の取り方・見出しの強弱・CTAの置き方・色のトーンなど）をルール化する。
   - 現在のコンセプト別テンプレート（minimal / corporate / warm / bold / elegant / modern）を、業種ごとに「どのテンプレ＋どのトーンが合うか」でマッピングし、必要なら業種専用のテンプレートを追加する。

2. **A（AIデザイナー）の使いどころ**
   - 「その店だけのキャッチコピー」「セクション見出しの言い回し」など**文言**の生成には向いている。
   - レイアウトや全体のビジュアルは、**テンプレート＋業種ルール**で固定し、色・フォントだけCSS変数で微調整する形にすると安定する。
   - 将来的に「AIがレイアウト案を提案し、人間が選んでからテンプレで組み立てる」ようにすることは可能だが、現状の「テンプレで3案出して選ぶ」方式の延長で実装するのが現実的です。

### 写真・背景について

- **ヒーロー・セクションの背景写真**は、**文字の入っていない画像**を選ぶと読みやすくまとまります。Google Maps の写真やデフォルト画像に文字が大きく入っていると、見出しと重なってごちゃごちゃしやすいため、可能なら文字なしのものを選んでください。

### 実装済みの内容

- **業種:** cafe / restaurant / salon / retail / **apparel** / service / **clinic** / general の8業種。AIの口コミ分析で conceptId を判定します。
- **業種ごとに6テンプレート:** 各業種で minimal / corporate / warm / bold / elegant / modern の6スタイルを用意（業種×スタイルで48種）。テンプレートIDは例: `cafe_minimal`, `apparel_elegant`。業種ごとの色・トーンの微調整（INDUSTRY_TWEAKS）を適用しています。
- **参照サイト収集:** キュー画面の「参照サイト（ウェブあり・上位表示分析）」で、**ウェブサイトがある店**を検索して保存できます。同じMaps検索で「ウェブあり」のみを取得し、順位目安（rankIndex）付きで保存します。
- **メタ取得:** 保存した参照サイトのURLから、サーバーで `<title>` と `<meta name="description">` を取得します。あわせて **HTML の断片を取得し、AI でデザイン特徴（色・フォント・レイアウト）を抽出**して各サイトに `designTraits` として保存します（Gemini API キーが設定されている場合）。
- **上位表示要因の言語化:** 保存した参照サイト一覧（文言＋デザイン特徴）を AI に渡し、「なぜ上位表示されているか」を **文言** と **デザインの共通項** の両方で言語化します。全体の傾向と業種・カテゴリ別の傾向を出し、テンプレート設計の参考にできます。数を増やしてから実行すると共通項がはっきりします。
