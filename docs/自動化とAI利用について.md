# 現在どこまで自動でできるか / 今後の拡張

## 現状（いま自動でできること）

| 処理 | 自動？ | 説明 |
|------|--------|------|
| ターゲット収集 | **手動トリガー** | 「キュー」で「検索」ボタンを押すと Google Maps で検索し、サイト未登録の店だけ表示。追加もボタン操作。 |
| 手動追加 | 手動 | Instagram 等で見つけた店をフォームで追加。 |
| 調査・LP生成 | **手動** | キューから「調査」を開き、コンセプト・強み・イメージカラーを**人が入力**。LPはテンプレートで生成（AI は使わない）。 |
| DM文面 | **手動** | ダッシュボードの「DM文面」欄に人が入力。 |
| 検閲 | 手動 | OK/NG は人がクリック。 |

**結論：いまは「寝ている間に勝手に調べて作成してメール文面まで用意」はできません。**

- すべて **ブラウザで人が操作** する前提です。
- バックグラウンドで動くサーバー・スケジューラ・バッチはありません。
- AI API も現状は **一切呼んでいません**（コスト削減のため）。

---

## 「寝ている間に全部やってほしい」にするには

次のような構成が必要です。

1. **常時動くサーバー（または FaaS）**
   - 例: Cloud Run / AWS Lambda / 自前サーバー で「定期実行」または「キュー監視」。

2. **自動実行の流れ（例）**
   - 毎日 2:00 に「港区 カフェ」で Maps 検索 → サイト未登録だけキューに投入。
   - キューに件数がたまったら、1件ずつ「調査」相当の処理（ここで AI を使うなら Gemini でコンセプト・強み・DM文案を生成）。
   - 生成した LP と DM 文面を DB やストレージに保存。
   - 朝、薫之介さんがダッシュボードを開くと「未検閲の案件」が並んでいる。

3. **AI を使う場合**
   - 調査・DM 文面を Gemini 等で生成するなら、その分の API 料金がかかります。
   - **金額上限** を守るには、サーバー側で「今月の利用金額（またはトークン数）を集計し、上限に達したら処理を止める」ロジックが必要です（下記「AI 金額上限」参照）。

現状のアプリは「全部ブラウザ内・手動」なので、上記は **別途バックエンド＋スケジューラ＋（必要なら）AI 連携** の設計・実装が必要です。

## AI の金額上限で止める

**現状**：アプリ内では AI API を呼んでいないため、**利用料・金額の概念がありません**。上限を「設定する場所」だけ用意している状態です。

**今後 AI（例: Gemini）を組み込む場合** の想定です。

- **クライアント（ブラウザ）だけ** で AI を呼ぶ場合  
  - 上限を「ブラウザの設定」で持つことはできるが、**改ざんや無視がしやすい**ので、厳密な「金額で止める」は難しい。
- **サーバーを用意する場合（推奨）**
  - サーバーで「今月の推定費用」や「トークン数」を集計する。
  - 設定した **月額上限（円）** に達したら、その月は AI を呼ばない（エラー返却や「上限到達」表示）。
  - 日次・月次でリセットするなど、集計単位を決めておく。

この「上限値の設定」と「上限に達したらストップする」は、**AI を呼び出す側（サーバー or クライアント）で、呼び出し前にチェックする**形で実装します。

アプリには「**AI 利用上限（円/月）の設定**」を保存する仕組みを追加できます。今は AI を呼んでいないので「設定を保存して表示するだけ」ですが、将来サーバーやクライアントで AI を呼ぶときに、この設定を参照して「上限に達したら処理を止める」ようにできます。

## フルオートにした場合の出費目安（最終チェックだけ人がやる想定）

**前提**：ターゲット収集 → 調査（AIで分析）→ LP生成 → DM文面生成 まで全部自動。人がやるのは「検閲ダッシュボードで OK/NG」だけ。

### 主な費用の内訳

| 項目 | 内容 | 月額の目安（円） |
|------|------|------------------|
| **Google Maps（Places API）** | エリア検索＋店舗詳細（サイト有無・住所など）。無料枠あり、超えると従量課金。 | 0〜約1.5万 |
| **AI（例: Gemini）** | 1店舗あたり「分析＋DM文案」で数百〜千トークン程度。Flash 系なら安い。 | 約500〜3,000 |
| **サーバー／FaaS** | 定期実行（cron）＋キュー処理。Cloud Run / Lambda など。 | 0〜約5,000 |
| **DB・ストレージ** | キュー・ダッシュボード用。Firestore 無料枠や小規模 Postgres など。 | 0〜約3,000 |

※ 為替・料金改定で変動します。あくまで目安です。

### 規模別のざっくり目安

- **小規模**（エリア1〜2、月50件前後）  
  **約 3,000〜8,000 円/月**  
  Maps 無料枠内＋Gemini 軽め＋サーバー最小構成。

- **中規模**（エリア複数、月 150〜300 件）  
  **約 1万〜2.5万 円/月**  
  Maps 少し超え＋Gemini 増＋サーバー常時 or 実行回数増。

- **大規模**（エリア多数、月 500 件以上）  
  **約 2万〜5万 円/月**  
  Maps 従量増＋AI・サーバー・DB すべて増。

### ポイント

- **Maps** が一番効きやすい。検索・詳細の回数と取得フィールドを絞るとコストを抑えられる。
- **AI** は Gemini Flash 系なら 1 店舗あたり 1 円未満〜数円程度のイメージで、件数が増えても比較的抑えやすい。
- **開発費** は別。フルオート化の設計・実装（バックエンド、スケジューラ、AI 連携、ダッシュボード連携）は初期コストがかかる。

## フルオートの使い方（実装済み）

1. **サーバーを起動**  
   `server/` で `npm install` → `.env` に `GOOGLE_MAPS_API_KEY` と `GEMINI_API_KEY` を設定 → `npm run dev` または `npm start`。
2. **フロントの .env**  
   `VITE_API_URL=http://localhost:3001` を設定して `npm run dev` でフロントを起動。
3. **設定タブ**  
   多言語・問い合わせフォーム・Instagram/LINE・Presented by・QRコードのオンオフをスイッチで設定。
4. **キュー**  
   「フルオート」で検索ワードを入力 → 「サーバーで収集」→「キューを1件処理」で、口コミ分析・LP3案・DM文面まで自動作成。
5. **検閲ダッシュボード**  
   承認待ちの案件だけ表示。LPは案1〜3を切り替えて確認し、DMを編集して OK/NG。

## ドメインについて

**ドメインの取得・DNS設定・ホスティングへの紐づけは、このアプリの外で行う必要があります。**

- このツールは「HTMLの生成」と「承認フロー」まで。生成したHTMLをどこに置くか・どのドメインで公開するかは、ご自身のサーバーやホスティング（Netlify、Vercel、自前サーバーなど）で設定してください。
- 生成HTML内の `canonical` や「このページのURL」を任意のドメインにしたい場合は、エクスポート後やサーバー側で URL を差し替えるか、設定項目として「公開用ベースURL」を将来追加する形になります。
