# 情報が省かれないようにする：情報量・遷移の拡張方針

## 方針の前提

- **「フォームを埋める」形式**  
  あらかじめ項目（枠）を決めておき、その枠を埋めていく形にする。後々の自動化（キュー処理・PDF取り込み・API連携）がしやすい。
- **一般的なウェブサイトに存在するもの／半々くらいで存在するものは、すべて標準枠に含める**  
  入れる場所を固定しておくことで「省かれる」を防ぎ、オプションで表示するもの（問い合わせフォーム・SNS・QRなど）も**枠として確実に盛り込む**。
- **この一覧にないものを追加したい場合**  
  有償オプションの**作り直し**を案内する。標準枠外の要望はカスタム対応とする。

---

## 課題（現状）

- **遷移ページがない**  
  すべて1ページ（LP）のみ。ナビは同一ページ内アンカー。
- **情報の「枠」が少ない**  
  生成時に作られるセクションは最大5つ（コンセプト・強み・雰囲気・お客様・アクセス）のみ。メニュー・営業時間・料金・FAQ・お問い合わせ本文などの**入れる場所が定義されていない**。
- **オプションが「後付け」になりがち**  
  問い合わせフォーム・SNS・QRなどはオプションとしてあるが、**標準の構成としての位置づけ**が弱く、フォームを埋める対象として明示されていない。

---

## 標準で含める項目一覧（フォームを埋める対象）

以下を「必ず枠として存在するもの」「オプションで表示するもの」に分け、すべてデータ構造とUIでフォームを埋める対象とする。

### 1. 常に枠として存在するセクション（本文エリア）

| id | タイトル例 | 想定内容 | 備考 |
|----|------------|----------|------|
| concept | コンセプト・こだわり | 店の理念・特徴 | ほぼ全サイト |
| menu | メニュー・サービス | 提供内容・品目 | ほぼ全サイト |
| hours | 営業時間 | 曜日・時間・定休 | ほぼ全サイト |
| access | アクセス | 住所・地図・交通 | ほぼ全サイト |
| price | 料金・プラン | 価格表・コース | 約半分。枠は常にあり、空可 |
| staff | スタッフ・私たち | 紹介 | 約半分。枠は常にあり、空可 |
| faq | よくある質問 | Q&A | 約半分。枠は常にあり、空可 |
| gallery | ギャラリー・写真 | 画像ブロック | 約半分。枠は常にあり、空可 |
| contact | お問い合わせ | 本文（「お気軽に」等の文言） | ほぼ全サイト |

- すべて**セクションとして枠を用意**し、生成・PDF取り込み・手入力で「埋める」。内容が無い項目は空または非表示でよい。
- ナビ・アンカーはこれらの id（`#concept`, `#menu`, `#access` 等）に合わせる。

### 2. オプションで表示するもの（チェックで ON/OFF、枠は常に存在）

いずれも「一般的なサイトにありがちなもの」として**標準枠に含め**、オプションで「表示するかどうか」を選ばせる。

| 項目 | 説明 | 既存実装 |
|------|------|----------|
| **問い合わせフォーム** | お問い合わせセクション内にフォームを表示。送信先URL（formActionUrl）を入力。 | ✅ contactForm, formActionUrl |
| **SNS・LINE 遷移** | Instagram / LINE へのリンク（フォロー・DM誘導）。専用セクションまたはフッター近くに表示。 | ✅ instagramLine, instagramUrl, lineUrl |
| **QRコード** | 店のURL等をQRコードで表示。掲載用・チラシ用。 | ✅ qrCode, qrCodeTargetUrl |
| **料金表セクション** | 上記の「料金・プラン」セクションを「表示する」かどうか。枠は常にあるが、未入力なら非表示にする場合はオプション扱い可。 | 要検討（表示/非表示トグル） |
| **Presented by** | フッターの「Presented by ウェブページ作成ツール」。削除は有償オプションの例。 | ✅ presentedBy（削除は有償） |

- **問い合わせフォーム・SNS・QR** は「オプションで発生するもの」として、**標準の構成に盛り込む**。データとしては常にフィールド（formActionUrl, instagramUrl, lineUrl, qrCodeTargetUrl）を持ち、オプションがONのときだけ該当ブロックを出力する。
- テンプレート・buildHtml では、これらのオプション用ブロックの**表示位置とスタイル**を標準で定義しておく（既存の extraSections の延長）。

### 3. その他・共通で持つ枠

- **ヒーロー**: headline, subheadline, ogImageUrl（トップ画像）
- **フッター**: footerText, footerAddress, footerPhone, footerEmail（2カラム等）
- **CTA**: ctaLabel, ctaHref（予約・お問い合わせボタン）
- **ナビ**: navItems（またはデフォルトで #concept, #menu, #hours, #access, #contact 等）

---

## 「この一覧にないもの」の扱い

- 上記の**標準枠・オプション以外**の要素（例：特定の外部予約システム埋め込み、独自レイアウト、追加ページなど）を希望する場合は、**有償オプションの作り直し**を案内する。
- 運用として「まずは標準枠で作成 → 足りない部分は有償で作り直し」とすることで、自動化しやすい範囲を固定しつつ、特別な要望には対応できる形にする。

---

## 実装まわりでやること（要約）

1. **セクション型の固定**  
   concept / menu / hours / access / price / staff / faq / gallery / contact を標準セクション型として定義。常にこの順で枠を用意（空可）。
2. **オプションの「枠」の明文化**  
   問い合わせフォーム・SNS・QR を「オプションで表示する標準ブロック」としてドキュメントとデータ構造に明示。既存の contactForm / instagramLine / qrCode はそのまま、標準構成の一部として扱う。
3. **生成・PDFでフォームを埋める**  
   analysisToContent や PDF 取り込みで、上記セクション型に対応するフィールドを可能な範囲で埋める。無い項目は空のまま枠だけ用意。
4. **ナビ・アンカーの統一**  
   実際に存在するセクション id に合わせてナビを生成（#menu, #access, #contact 等）。オプションで SNS セクションがある場合は #sns 等を追加可能に。

---

## まとめ

- **一般的なウェブページに存在するもの・半々くらいで存在するものは、すべて標準枠に盛り込む。**
- **問い合わせフォーム・SNS遷移・QRコードなどオプションで発生するものも、標準の「枠」として確実に盛り込み、フォームを埋める形式にする。**
- **枠を固定してフォームで埋める形にすることで、後々の自動化がしやすい。**
- **この一覧にないものを追加したい場合は、有償オプションの作り直しを選んでもらう。**
