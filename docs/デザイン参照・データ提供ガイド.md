# デザイン参照・データ提供ガイド

AI（実装時の指示・CSS方針）が**後から参照しやすい**形でのデータの渡し方です。

## 結論：丸ごとHTML（Wix等）だけで十分か？

| 手段 | メリット | デメリット |
|------|----------|------------|
| **URLのみ** | 手軽 | こちらが常に同じ見え方で開けるとは限らない。コードは読めない。 |
| **巨大HTML＋Wix/CSS一括貼り** | 色変数（`--color_*`）、フォント指定、余白の傾向は**断片的に**拾える | **ノイズが極大**（Thunderbolt・minify・数千行の基盤CSS）。「再現用レシピ」には不向き。 |
| **URL ＋ 短いメモ（推奨）** | 意図が伝わる | ユーザー作業が少し増える。 |
| **スクショ ＋ URL** | レイアウト・階層・雰囲気の共有に最強 | 数値は手で書く必要あり。 |

**おすすめ順**  
1. **URL**  
2. **「真似したい点」を3〜7行**（例：ヒーロー全幅＋ロゴオーバーレイ、見出しは英字大＋和文小、セクション間 120px 前後、ボタンは線のみ など）  
3. **任意：デスクトップ／スマホのスクショ各1枚**  
4. **任意：DevToolsで拾ったトークンだけ**（背景 `#f5f3ef`、見出しフォント名、アクセント色 など）

数十サイト分も**フルHTMLを貼るより**、上記の組み合わせの方が、実装方針に翻訳しやすく、ミスも減ります。

### 「いいと思ったポイント」を番号付きで書く形式について

**その形式で問題ありません（推奨に近いです）。**

- URL ＋ **自分の言葉で箇条書き（1. 2. 3. …）** → 何を優先すべきかが一目で分かる。
- スクショがあると **「世界観」「余白の取り方」** の意図がさらに伝わる（任意だが有効）。

---

## 注意（著作権・商用）

- 参照は**パターン・比率・雰囲気**の学習用。ロゴ・写真・独自コピー・特徴的な装飾の**丸コピは避ける**前提です。
- Wix出力は**プラットフォーム著作物＋クライアント素材**が混ざるため、「このHTMLをそのまま流用」は不可。

## 記述例（そのまま真似できる）：44APARTMENT

**URL:** [https://www.44apartment.com/](https://www.44apartment.com/)  
**業種:** カフェ／レストラン（複数店舗）

**いいと思ったポイント（ユーザー記述・実装方針の種になる）：**

1. **ファーストビュー** — URLを開くとまず画面一面に写真。数秒おきに写真が切り替わり、店名で世界観が最初から伝わる。
2. **タイポと余白** — シンプルで素朴、伝わりやすい文字サイズ・フォント・細さ。写真・余白・文章のバランスがよい。
3. **複数店舗** — ただ遷移するだけでなく、店舗ごとに情報と写真がある。Google マップは**サイト内埋め込みではなく外部へ遷移**し、世界観を壊さない。
4. **メニュー** — カフェ／レストランでメニューが写真中心の場合、**1からHTMLメニューを作らず**、メニュー画像やPDF等へのリンクに任せる工夫。

（HTML全文は不要。上記＋任意でモバイル／PCのスクショ数枚で十分。）

## 記述例：ゆずの木 ねむの木 みずたまの木（予約ページ）

**URL:** [https://www.mizutama-cafe.com/reservation](https://www.mizutama-cafe.com/reservation)  
**業種:** カフェ（完全予約制・自然系）

**いいと思ったポイント（カフェ参照・Warm Organic 寄り）：**

1. **写真と説明・情報の見せ方** — 世界観が伝わりやすい大きな写真と、簡潔な説明。提供時間などは**文字色を強調するのではなく、背面に優しい色の帯**（森の写真に合うベージュ系など）を敷くことで、**目に優しく・読みやすさ**と両立している。
2. **SNSリンク** — LINE・Instagram などを**「文字リンク」ではなくアイコン**で出しており、直感的でスマートに見える。
3. **ナビゲーション** — **サイドタブ（ハンバーガー／ドロワーメニュー）**で主要ページへまとめてアクセスし、**1ページあたりのスライド（カルーセル枚数）を増やさない**整理の仕方になっている。

（ポイント数はサイトごとに 3 件でも 5 件でも可。上記のように番号付きで書けば十分。）

## 記述例：Dans Dix ans（ダンディゾン）※パン屋・Warm Organic 参照

**URL:** [https://dansdixans.net/](https://dansdixans.net/)  
**業種:** パン屋（ベーカリー）— カフェ系テンプレ `warm_organic` の**商品一覧・世界観**の参照例として有効。

**いいと思ったポイント：**

1. **背景と商品写真・タイポ** — 背景を**一色で統一**し、パン写真に**背景透過**があることで「地続きで一覧が続く」わかりやすい印象にしつつ、**文字ブロックで区切り**を付けている。**商品名と説明文**で**色・サイズ（強弱）**を変え、のっぺりとした単調さを避け、読みやすくしている。
2. **周辺UIの統一** — **カレンダー**など予約・情報系の部品も、サイト全体の**世界観（色・線・フォントのトーン）**に揃えている。
3. **手書きイラスト** — 手描きイラストを使う場合は、**そのサイトのトーン**（素朴・ナチュラル等）に**合わせる**と一貫した印象になりやすい、という示唆。
