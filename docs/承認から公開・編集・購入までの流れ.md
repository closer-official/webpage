# 承認から公開・編集・購入までの流れ

## OK / NG ボタンについて

- **OK** … その案件を「承認済み」にして、**OK済み — メール送信と確認**のリストに移すだけです。**メールやLPの送信は行われません。** 送信するには、OK済みになったあとで「メール送信（mailto）」や「送信済み」を押す／自分で送る操作が必要です。
- **NG** … 「却下」にし、一覧からは表示されなくなります。データは残りますが、画面には出てきません。

---

## 共有用URL（スマホ・他端末でLPを開く）

**「このページを別タブで開く」** で開くのは **blob URL** のため、**その端末・そのセッションでしか見られず**、URLをコピーしてスマホに送っても開けません。

そこで **共有用URL** を用意しています。

1. **検閲ダッシュボード**で、各案件のLPプレビュー欄に **「共有用URL（スマホ・他端末で開けます）」** が表示されます（APIサーバーに接続しているときのみ）。
2. そのURLをコピーして、**スマホや別のPCのブラウザで開く**と、同じLPが表示されます。
3. **本番（Vercel など）にデプロイした環境**では、そのURLを誰に送っても開けます。**ローカル開発**では、同じPC内でしか有効になりません。

共有用URLは **`/api/preview/案件ID`** の形です。サーバーがその案件のHTMLをその場で組み立てて返しているため、どこから開いても同じ内容が見られます。

---

## LPのメニューに「購入」ボタン（Stripe 決済）

**Stripe を有効にしているとき**、共有用URLで開いたLPの **メニュー一覧**（こだわり・メニュー・アクセスの並び）に **「購入」** が追加されます。

- 店や見学者が **「購入」** を押すと、**Stripe Checkout** に飛び、設定タブの **料金・お支払い** で決めたプラン・オプションの金額で支払いができます。
- **決済が完了**してLPに戻ると、**「購入」のリンクは非表示**になります（同じURLに `?payment=success` が付いて戻るため）。

※ 購入ボタンが付くのは **共有用URL**（`/api/preview/:id`）で表示したLPのみです。「このページを別タブで開く」（blob）やダウンロードしたHTMLには、決済用の仕組みは入りません。

## 店が「編集したい」「購入したい」と言ったとき

### 1. 編集してほしい場合

- **共有用URL** を店に送り、内容を確認してもらう。
- 修正したい点（文言・写真・営業時間など）を聞く。
- こちらで **調査画面（手動・詳細）や検閲ダッシュボード** から該当案件を開き、**コンセプト・強み・セクション文・SEO** などを編集。
- 編集後、再度 **HTMLを生成**（案の差し替えや再ビルド）し、**共有用URL** または **「このページを別タブで開く」** で確認。
- 問題なければ、**HTMLをダウンロード** して自前でホスティングするか、次の「購入・公開」の流れへ。

**「調査／ダッシュボードで文言・SEO等を編集」とは**

- **調査画面（手動・詳細）** … キューに追加した店を「調査」するタブ。ここで **テンプレート**・**コンセプト**・**強み**・**セクション（メニュー・営業時間・アクセスなど）の本文**・**ナビ項目** などを編集し、「LPを作成」でHTMLを組み直します。フルオートで作った案件は一度キュー経由で開くか、ダッシュボード側で直します。
- **検閲ダッシュボード** … フルオートやキュー処理で作られた案件が並ぶ画面。各カードの **LPプレビュー** と **DM文面** を確認し、**OK／NG** を付けます。ここでは **DM文面のテキスト** をその場で編集できます。LPの見出し・セクション文・SEO（タイトル・説明文）を変えたい場合は、**調査画面** で該当店を開くか、**PDF・テキストから作成** や **Export** で同じテンプレに差し替えて編集する流れになります（ダッシュボード上でセクション文を直接編集する機能は現状ありません）。

### 2. 購入・公開したい場合（料金を取って納品する場合）

1. **料金の確定**  
   - 設定タブの **料金・お支払い** で、プラン（通常／学割など）とオプションを選択。  
   - **「料金を再計算」** で合計を出し、店に提示。

2. **Stripe で決済**  
   - 「Stripeで支払う」で Checkout に飛ばし、店（または自分）が支払い。  
   - 決済完了後、**Webhook** または **success_url 戻り** で「購入済み」を記録（実装方針に依存）。

3. **納品の形**  
   - **A. 共有用URLのまま使ってもらう**  
     本番デプロイ済みなら、共有用URL（`https://あなたのドメイン/api/preview/案件ID`）をそのまま店に渡す。**無料のまま見せる・テスト用**ならこれで十分。  
   - **B. HTMLを渡して自社で公開してもらう**  
     検閲ダッシュボードで「このページを別タブで開く」→ ブラウザの「名前を付けて保存」でHTMLを保存し、それを渡す。店が自社サーバー・Netlify・レンタルサーバーなどにアップロード。  
   - **C. こちらのサーバーでホスティング**  
     未実装。将来、OKした案件のHTMLを S3 / Netlify / Vercel などに自動アップロードし、**店専用の公開URL** を発行する機能を足す想定（`docs/仕様と制限・Stripe決済.md` の「2-6. 承認済みLPのホスティング・公開」参照）。

4. **ドメイン・SSL**  
   - ドメイン取得・DNS・SSL は **このアプリの外** で行います。店が自分で持っているドメインに載せるか、あなたが代行する場合は別途手順が必要です。

### 3. まとめ

| 店の要望           | 対応の目安                                                                 |
|--------------------|----------------------------------------------------------------------------|
| 内容を直したい     | 調査／ダッシュボードで編集 → 再生成 → 共有用URLで確認 → 必要ならHTML納品 |
| 料金を払って欲しい | 料金・お支払いで金額確定 → Stripe決済 → 共有URL or HTMLで納品            |
| 自分で更新したい   | 現状はHTML編集 or 将来の「管理者画面（CMS）」オプション（有償）を案内     |
