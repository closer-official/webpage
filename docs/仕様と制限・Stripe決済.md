# 現在の仕様でできないこと・必要な対応

## 1. できないこと一覧

| 項目 | 現状 | 備考 |
|------|------|------|
| **寝ている間の完全自動実行** | 本番は制限あり | 本番では Vercel Cron で自動実行可能。Hobby プランは **1日1回まで**（現在は UTC 0:00）。毎分実行したい場合は Pro プランが必要。 |
| **ドメイン取得・DNS・ホスティング** | 不可 | アプリ外。取得・設定は手動で行う必要がある。 |
| **多言語LPの実際の出力** | 未実装 | オプションの「多言語対応」はON/OFFのみ。他言語のHTMLは未生成。 |
| **QRコード画像の生成** | 実装済み | オプションで `qrCodeTargetUrl` を指定するとそのURLのQR画像をLPに埋め込む。 |
| **問い合わせフォームの送信先** | 実装済み | オプションで `formActionUrl` を指定するとLPのフォームの action に設定される。 |
| **Instagram/LINE URL の自動取得** | 不可 | Maps API からは取得できない。手動でメモに入れるか、別サービスで補う必要がある。 |
| **AI利用額の実計測と上限で停止** | 未実装 | 上限「円/月」は保存するが、Geminiの利用額を集計しておらず、超えても止まらない。 |
| **承認済みLPのホスティング・公開** | 一部実装 | **共有用URL**（`/api/preview/:id`）でスマホ・他端末からLPを開ける。恒久公開はHTMLダウンロード後にNetlify/Vercel等へ手動デプロイ。 |
| **ユーザー認証・複数ユーザー** | なし | キュー・ダッシュボード・オプションは1サーバーで1つ。ログインや権限なし。 |
| **オプションに応じた課金・決済** | 実装済み | 料金算出API・Stripe Checkout は実装済み。`STRIPE_SECRET_KEY` で利用可能。 |

**標準枠外の追加について**  
LPに含める項目（コンセプト・メニュー・営業時間・アクセス・料金・お問い合わせ・問い合わせフォーム・SNS・QR など）は「標準で含める項目」として枠が決まっている。この一覧に**ない**要素（独自レイアウト・外部予約埋め込み・追加ページなど）を希望する場合は、**有償オプションの作り直し**を案内する。詳細は `docs/情報量・遷移の拡張方針.md` を参照。

## 2. できるようにするために必要なこと

### 2-1. 寝ている間の自動実行

- **やること**  
  - サーバーに cron 相当を入れる（例: `node-cron` で毎日2時など）。  
  - または Cloud Scheduler / GitHub Actions などで `POST /api/process-next` を定期呼び出し。  
- **注意**  
  - キューが空のときは何もしない。  
  - Maps/Gemini の利用量・コストに上限をかける処理を入れた方が安全。

### 2-2. 多言語LPの実際の出力

- **やること**  
  - オプション「多言語対応」ON のとき、LP生成時に言語を複数指定（例: 日本語・英語）。  
  - 各言語用のテキストを Gemini で翻訳するか、固定文言を用意。  
  - `buildHtml` で言語ごとのHTMLを生成するか、1HTML内で `lang` 切替。  
- **注意**  
  - 翻訳する場合はトークン料金が増える。  
  - 料金に多言語分を反映するなら、後述の料金計算に「多言語」を項目として入れる。

### 2-3. QRコード画像の生成

- **実装済み** … オプションで「QRコードを発行して掲載」をONにし、`qrCodeTargetUrl` にURLを指定すると、そのURLのQR画像をLPに埋め込む。

### 2-4. 問い合わせフォームの送信先

- **実装済み** … オプションで「問い合わせフォームを設置」をONにし、`formActionUrl` に送信先URL（または `mailto:...`）を指定すると、LPのフォームの action に設定される。

### 2-5. AI利用額の実計測と上限で停止

- **やること**  
  - Gemini を呼ぶたびに、入出力トークン数（または公式の単価）から円換算し、DBやJSONに「今月の利用額」を加算。  
  - 「キューを1件処理」の前に、設定した月額上限と比較し、超えていれば処理せずエラー返却。  
  - 月初で利用額をリセット（月次集計）。  
- **注意**  
  - 単価は Gemini の料金表に依存する。  
  - クライアントの「AI利用上限」設定をサーバーに渡すか、サーバー側でも同じ上限を保持する。

### 2-6. 承認済みLPのホスティング・公開

- **現状**  
  - **共有用URL** … `GET /api/preview/:id` で案件のLPをHTMLとして返す。検閲ダッシュボードに「共有用URL」を表示し、コピーしてスマホ・他端末で開ける。本番デプロイ後は誰でもアクセス可能。  
  - **恒久公開** … 未実装。OKした案件のHTMLをストレージ（S3 / GCS）へアップロードしたり、Netlify/Vercel の API でデプロイする機能はなく、HTMLダウンロード後に手動でホスティングする。  
- **将来やること（恒久公開をアプリで行う場合）**  
  - 公開URLをダッシュボードに表示し、必要ならQRコードの生成URLにも使う。  
  - ドメイン・SSLはホスティング側の設定。デプロイを「有料オプション」にするなら、Stripeの商品・価格に含める。

### 2-7. ユーザー認証・複数ユーザー

- **やること**  
  - 認証（例: Firebase Auth / Auth0 / 自前JWT）を導入。  
  - キュー・ダッシュボード・オプションを「ユーザーID」で分離。  
  - APIを「ログイン済みユーザーに紐づくデータだけ」返すように変更。  
- **注意**  
  - Stripe と連携する場合、`customerId` や `subscriptionId` をユーザーごとに保存する必要がある。

## 3. オプションで金額を算出し Stripe で決済する

### 3-1. やりたいこと（イメージ）

- オプション（多言語・問い合わせフォーム・SNSリンク・Presented by・QRコードなど）を選択する。  
- 選択内容に応じて**金額が自動計算**される。  
- 「支払う」で **Stripe Checkout** に飛び、決済完了後にそのオプションを有効にする（または1回限りの利用権として記録する）。

### 3-2. 必要な作業の流れ

1. **料金体系の定義**
   - 例: 基本料金（1案件あたり）＋ オプション単価。
   - 例: 基本 500円、多言語 +300円、問い合わせフォーム +200円、QRコード +100円、など。
   - これをサーバー側の「料金表」として持つ（定数またはDB）。

2. **サーバー: 料金計算API**
   - `POST /api/price` などで、選択オプション（`multiLanguage`, `contactForm`, `qrCode` など）を受け取る。
   - 料金表に基づいて合計金額（円）を計算し、`{ amountYen, items: [...] }` のように返す。

3. **サーバー: Stripe Checkout Session 作成**
   - Stripe の SDK を入れ、`STRIPE_SECRET_KEY` を環境変数に設定。
   - `POST /api/create-checkout-session` で、上記と同じオプションと金額を受け取る。
   - `stripe.checkout.sessions.create` で `line_items`（金額は円→ Stripe は最小単位で指定するため、日本なら円のまま amount を指定可能）、`success_url` / `cancel_url`、`metadata` にオプションを保存。
   - レスポンスで `session.url` を返す。

4. **フロント: 料金表示と「支払う」**
   - 設定タブ（または専用の「プラン・お支払い」画面）で、オプションのON/OFFに応じて `POST /api/price` を呼び、**金額を表示**。
   - 「Stripeで支払う」ボタンで `POST /api/create-checkout-session` を呼び、返ってきた `session.url` に `window.location.href` で遷移。

5. **決済完了後の扱い**
   - **A. チェックアウト成功ページ**  
     `success_url` で戻す（例: `/?payment=success`）。フロントで「決済完了。オプションが有効です」と表示。  
   - **B. Webhook（推奨）**  
     Stripe の `checkout.session.completed` をサーバーで受信。`metadata` からオプションを読み、DBやストアに「このユーザー（またはセッション）はこれらのオプションを購入済み」と保存。  
   - その後、LP生成時に「購入済みオプション」を参照して有効にする。

6. **オプションの「有効」の持ち方**
   - 現状は「全員同じオプション」なので、まずは「直近の決済が成功したオプションをストアに保存」でよい。
   - ユーザー認証を入れたら、「ユーザーごとの購入済みオプション」として保持する。

### 3-3. 実装時の技術メモ

- **Stripe の鍵**  
  - 本番: `STRIPE_SECRET_KEY`（サーバーのみ）、`STRIPE_PUBLISHABLE_KEY`（フロントで Stripe.js を使う場合）。  
  - テスト: `sk_test_...` / `pk_test_...` で同じ流れを試せる。

- **金額**  
  - Stripe は日本円なら `amount: 500`（500円）のように指定可能。  
  - 計算はサーバーで行い、クライアントに渡す金額で Checkout を作成する（クライアント任せにしない）。

- **Webhook**  
  - `stripe listen --forward-to localhost:3001/api/webhooks/stripe` でローカル受信。  
  - 本番では Stripe ダッシュボードでエンドポイントURLを登録し、`webhookSecret` で署名検証する。

- **CORS**  
  - Checkout はリダイレクトなので、通常のAPIと同じCORSでよい。  
  - `success_url` / `cancel_url` は自サイトのパスでよい。

## 4. まとめ

- **できないこと** … 上記一覧のとおり（ドメイン・多言語出力・AI額集計・LP自動デプロイ・認証）。自動実行は本番で Cron 対応（Hobby は1日1回）。問い合わせフォーム送信先・QRコード・課金・Stripe は実装済み。
- **できるようにするには** … 各項目で「2. 必要なこと」の対応が必要。
- **オプションで金額算出＋Stripe決済** … 料金表の定義 → 料金計算API → Checkout Session 作成 → フロントで金額表示とリダイレクト → Webhook で購入内容を保存、の順で実装すれば実現できる。

次のステップとして、**料金表（基本料金・オプション単価）を決める**と、Stripe 連携の実装範囲が明確になります。

## 5. 実装済み: 料金算出と Stripe 決済

- **料金表** … `server/price.js` で定義。変更時はこのファイルを編集する。

| 種別 | 項目 | 金額 | 備考 |
|------|------|------|------|
| **プラン** | 通常プラン | 39,800円 | 一般の個人店・事業主 |
| | 学割プラン | 19,800円 | 学生起業家・フリーランス |
| **紹介** | 紹介コード（基本料金無料） | 通常／学割の**基本料金のみ0円** | コードは**紹介者ごとに Supabase `referral_codes` へ1行登録**（`code` は任意の文字列で可）。オプション・その他サービスは別途課金。合計は0未満にならない（`Math.max(0, 合計)`）。 |
| **削除** | 言語の削除（1言語につき） | ▲2,000円 | 不要な言語ファイルを消す |
| | お問い合わせフォームの削除 | ▲3,000円 | 予約は電話かインスタDM向け（▲5,000円は要拡張） |
| | SNSフィード連携の削除 | ▲2,000円 | 更新が止まっている店向け |
| | 簡易マップ表示の削除 | ▲2,000円 | 店舗を持たないブランド向け |
| **追加** | Presented by 削除 | +5,000円 | フッター1行削除 |
| | カスタマイズQRコード作成 | +2,000円 | ツールで生成して渡す |
| | Webクーポン設置 | +5,000円 | コンポーネント1つ追加 |
| **その他** | 店名.store-official.net 提供 | 5,000円 | `店名.store-official.net` で公開する場合 |
| | 独自ドメイン取得・運用（1年単位） | 5,000円/年 | 初回に希望年数分を一括支払い。返金なし |
| | 管理者画面（店専用CMS）追加 | 20,000円 | **顧客がオプション購入時**に、その店だけ使える管理画面を用意。閲覧数の確認、LP内の**文章・写真の差し替え**ができる想定（契約後に実装・セットアップ）。 |
| | オンライン決済導入 | 30,000円 | Square / Stripe連携 |
| | デザインフルカスタム | 50,000円〜 | 1からこだわりたい方向け |
| | SEO・MEOセット | 20,000円 | Google検索・マップ最適化 |
- **ドメイン運用ルール（確定）**
  - 無料提供時は `店名.closer-official.com` を基本とし、`.web.app` は使わない。
  - 独自ドメイン申込時は、候補を **上位5つ**受け取る（前半部分 2〜3案 + 末尾部分 2〜3案）。
  - 期限前通知は **半年前 / 3か月前 / 1か月前** の3回メールで実施。
  - 更新意思と支払いが確認できた場合のみ継続。未確認・未払いの場合は期限到来で停止。
  - 倒産・事業終了などを含め、契約期間中の中途解約は返金しない。
- **API**  
  - `GET /api/billing` … 保存済みのプラン・オプションを返す。  
  - `POST /api/billing` … プラン・削除/追加オプションを保存。  
  - `GET /api/price-plans` … プラン一覧・削除/追加/その他オプションの定義を返す。  
  - `POST /api/price` … 請求選択（billing）を body で送ると合計金額と内訳を返す。  
  - `GET /api/stripe-configured` … `STRIPE_SECRET_KEY` が設定されているか返す。  
  - `POST /api/create-checkout-session` … body に `billing`, `successUrl`, `cancelUrl`。Stripe Checkout の URL を返す。
- **フロント** … 設定タブに「料金・お支払い」を表示。プラン（**通常 / 学割**）を選択し、紹介コード（任意）・削除オプション・追加オプション・その他サービスを指定。選択に応じて合計が変わり、「料金を再計算」で反映。「Stripeで支払う」で Checkout にリダイレクト。決済後は `?payment=success` で戻る。
- **サーバー環境変数** … `server/.env` に `STRIPE_SECRET_KEY=sk_...` を設定すると決済が有効になる。テスト時は `sk_test_...` を使用する。**紹介コード**は本番で Supabase テーブル `referral_codes` を使用（マイグレーション `20250316120000_referral_codes.sql`）。**Supabase 未接続のローカル開発**では、暫定で `REFERRAL_CODES=コードA,コードB`（カンマ区切り）に一致するコードのみ有効。
- **合計0円** … 紹介で基本料金のみ無料かつオプションなしのとき、Stripe は使わず API が `{ free: true, url: null }` を返す。お客様への最終案内（メール／LINE・紹介コードの手渡し・紹介者への還元など）は**手動**で行う。

### 5-2. LP 埋め込み用「料金・お支払い」セクション（不正利用防止）

- **対象** … イベント用テンプレ（`event`）以外の**全テンプレ**の生成LPに、「料金・お支払い」をメニューとセクションとして追加する。
- **目的** … 支払いが完了するまでページに料金・お支払いを表示し続け、未払いのままサイトを利用している不正を防ぐ。
- **挙動**  
  - ナビに「料金・お支払」リンク（`#payment`）を追加。  
  - セクション `#payment` に iframe で「料金・お支払い」フォームを表示。フォーム内でプラン・オプションのON/OFF、金額算出、「支払いを確定する」で Stripe Checkout へ遷移（同一タブ）。決済後は LP の URL に `?payment=success` 付きで戻る。
- **API** … `GET /api/lp-payment-form?returnUrl=<LPのURL>` で埋め込み用フォームの HTML を返す。iframe の `src` はビルド時に `data-src` で渡し、表示時に `returnUrl` を付与して読み込む。
- **genOptions**  
  - `paymentFormBaseUrl`（任意）… フォームを配信するオリジンのベース URL。未指定時は同一オリジン（`/api/lp-payment-form`）。LP を別ドメイン（例: `店名.store-official.net`）に公開する場合は、API サーバー（例: `https://closer-official.com`）を指定する。
- **Stripe の利用** … サーバーに `STRIPE_SECRET_KEY` を設定するだけで、埋め込みフォームからも同一の `/api/price`・`/api/create-checkout-session` が使え、決済と機能利用が可能になる。
