# テンプレ制作 — オーナー用（やること一覧）

**あなた（リポジトリ責任者）がテンプレ作業の外側でやることだけ**を並べた短冊です。  
**契約・実装の中身**は必ず `docs/テンプレ制作_必須契約_AI向け.md` を正とする。

---

## 作業が始まる前

- [ ] GitHub（等）で **作業者にリポジトリアクセス**を付与した（Collaborator / Org メンバー）。
- [ ] 作業者に **`docs/テンプレ制作_必須契約_AI向け.md` を読ませる**（または Cursor で `@` させる）よう伝えた。
- [ ] 必要なら **`docs/テンプレ制作ワークスペース要件書.md`** も渡した（ギャラリー掲載か・納品レーンかの整理用）。

---

## プルリクエストが来たら（レビュー）

- [ ] **ブランチが `main`/`master` 直 push ではなく PR になっている**。
- [ ] 必須契約 **`テンプレ制作_必須契約_AI向け.md` の「2. 実装の必須チェックリスト」**（A〜I、固定 HTML 型なら J）を、差分と照らして確認した。
- [ ] ローカルまたは CI で **`npx tsc --noEmit`** が通ることを確認した（または CI を信頼してグリーンであること）。
- [ ] **`public/deliverables/**` の既存ファイルに、依頼のない変更が混ざっていない**（納品物保護ルール）。

---

## マージしたあと

- [ ] **本番／プレビュー環境のデプロイ**が完了した（通常は Vercel の自動デプロイ。チーム手順に従う）。
- [ ] **`/api/template-preview/<templateId>`** 等で、追加テンプレの表示を目視した（可能なら）。
- [ ] **顧客にギャラリーを見せない方針**なら、`galleryDraftBuiltins` や公開 API の運用が意図どおりか確認した（必須契約・要件書参照）。

---

## 作業者に Cursor でやらせるとき（依頼文の最小セット）

次をそのまま送れば足りることが多いです。

1. リポジトリの **clone URL**  
2. 「新規 Cursor チャットの **1 通目**で `@docs/テンプレ制作_必須契約_AI向け.md` を添付してから要件を書く」  
3. 「終わったら **PR** を出す。マージはオーナー側」

---

## 迷ったら

- **実装の細部・ファイル名** → `テンプレ制作_必須契約_AI向け.md`  
- **ギャラリーと納品の違い・Vercel** → `テンプレ制作ワークスペース要件書.md`  
- **どのフォルダに何があるか** → `テンプレまわりの置き場とデプロイ.md`
