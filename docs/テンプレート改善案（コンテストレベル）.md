# 各テンプレート：コンテスト受賞レベルに向けた不足要素の言語化

**目的**: ウェブサイトのコンテストで受賞できるレベルまで各テンプレートを仕上げる。  
**流れ**: 言語化 → 承認 → 実装（本ドキュメントは「言語化」のみ。実装は承認後に実施）

---

## 前提：コンテスト審査で見られがちな観点

- **ビジュアル階層**：どこを先に見せるかが明確か
- **タイポグラフィ**：フォントの選び方・サイズリズム・行間
- **余白とグリッド**：意図のあるスペーシング
- **CTA（行動喚起）**：予約・問い合わせなど「次にやること」が分かるか
- **ナビゲーション**：ヘッダー／フッターがその世界観に合っているか
- **画像・ビジュアル**：質・構図・テンプレートとの一貫性
- **インタラクション**：ホバー・フォーカス・スクロール時の動き
- **フッター**：1行テキストではなく、構成されたフッターか
- **レスポンシブ**：モバイルで崩れていないか・タッチしやすいか
- **アクセシビリティ**：コントラスト・フォーカス表示・見出し構造

---

## 全テンプレート共通で足りていない要素

| 観点 | 現状 | 不足しているもの |
|------|------|------------------|
| **ヘッダー** | ロゴ＋リンクのみ。ナビはなし | 各世界観に合ったナビ（メニュー／予約ボタン等）。スクロール時の挙動（固定・透過→塗りなど）の定義 |
| **プライマリCTA** | ヒーロー直後に明確なボタンがない | 「予約する」「お問い合わせ」「無料相談」など、1ページに最低1つの主役ボタンとそのスタイル |
| **フッター** | 1行テキスト＋Presented by のみ | 住所・電話・SNS・リンク・コピーライトなど、2〜3カラム程度の「フッター構成」とスタイル |
| **セクションの種類** | 見出し＋本文の1パターン中心 | 画像付きセクション・引用（pullquote）・数字で見せるブロック・ギャラリーなど、バリエーション |
| **リンク・ボタンのホバー／フォーカス** | ほぼ未定義 | 各テンプレートのトーンに合ったホバー・フォーカス可見の指定 |
| **スキップリンク** | なし | アクセシビリティ用「メインコンテンツへ」のスキップリンク |

---

## A-1：Minimal Luxury（超高級ホテル・エステ）

**現状あるもの**: オフホワイト背景、Playfair Display、非対称ヒーロー、角丸なし、スクロールフェードイン、py-32 の余白

| 観点 | 不足している要素（言語化） |
|------|---------------------------|
| **ナビ** | ロゴ＋「宿泊」「エステ」「アクセス」「予約」程度の最小限メニュー。ヒーロー上では背景に溶け、スクロールで線や背景が出るような挙動 |
| **ヒーロー直後** | 「ご予約はこちら」「お問い合わせ」など、セリフ体・アウトラインまたは最小限の塗りボタン。ホバーでわずかに透明度やボーダーが変わる程度 |
| **セクションの幅** | コンセプト文だけではなく、「体験・メニュー」を画像＋短テキストのカード（aspect-ratio 3/4 維持）で並べるブロックがあると良い |
| **引用** | 雰囲気づくりの1行引用（例：「時間を、手放す。」）を入れるブロックとスタイル（小さめ・イタリック・字間） |
| **フッター** | ロゴまたはブランド名・住所・電話・メール・SNS を小さめのキャプスで。区切りは細い線 1px、余白は 32/64 を維持 |
| **画像** | ヒーロー以外に、セクション用の画像スロット（1〜2枚）があると「高級感」の訴求が増える |
| **インタラクション** | リンク・ボタンのホバーは「下線が伸びる」「opacity 変化」など、控えめな動き |

---

## A-2：Dark Edge（漆黒とインパクト）

**現状あるもの**: 全画面ヒーロー、背景固定、mix-blend-difference、uppercase

| 観点 | 不足している要素（言語化） |
|------|---------------------------|
| **ナビ** | 仕様の「全画面 z-50 メニュー」が未実装。ハンバーガー→開くと全画面オーバーレイでメニュー項目＋RESERVE 等。閉じると再びアイコンのみ |
| **ヒーロー内CTA** | 「RESERVE」「ACCESS」など、枠線だけのボタンまたは反転ボタン。ホバーで fill するか、色が反転する動き |
| **本文セクション** | 今は白文字＋黒背景のフラットなブロック。ところどころ full-bleed の画像や、細いアクセント線（金や白）で区切るバリエーションがあると良い |
| **アクセント色** | 全体はモノクロに近いが、CTA や重要な語だけに「金」「赤」などをごく限定的に使うと審査で差がつきやすい |
| **フッター** | 黒背景のまま、小さい uppercase で RESERVE / CONTACT / 住所。線は #222 程度で |
| **スクロール** | ヒーローを過ぎたあとのセクションが視界に入るとき、わずかにフェードや y 移動があると「映画っぽさ」が増す |

---

## A-3：Corporate Trust（重厚と信頼）

**現状あるもの**: 3カラム・border-left アクセント・Ken Burns 風アニメ

| 観点 | 不足している要素（言語化） |
|------|---------------------------|
| **ヒーロー** | 現状は中央テキストのみ。背景にオフィスや建物の画像＋オーバーレイ、または左にキャッチ・右に画像の2カラムがあると「会社」感が出る |
| **数字・実績** | 「25年」「○○社」など、数字を大きく見せるセクション（1行 or 2〜3カラム）。フォントは sans のまま、サイズだけ強く |
| **プライマリCTA** | 「資料請求」「無料相談」など、塗りボタン（青系）をヒーロー直下またはナビ右に。ホバーで少し暗くする |
| **ナビ** | ロゴ＋「サービス」「実績」「会社概要」「お問い合わせ」＋CTA ボタン。線で区切ったシンプルなヘッダー |
| **フッター** | 2〜3カラム：会社名・住所・電話／サービスリンク／お問い合わせ。border-top で区切り、小さめのテキスト |
| **カード** | 3カラムのカードに、ホバーで border-left が太くなる or 影がつくなど、軽いインタラクション |
| **セクション画像** | ヒーロー以外に、オフィスやチームの画像を1枚入れるブロックがあると信頼感が増す |

## B-1：Warm Organic（自然と調和）

**現状あるもの**: 角丸 40px、dashed、温色、hover scale

| 観点 | 不足している要素（言語化） |
|------|---------------------------|
| **画像の使い方** | 「あえて rotate」はあるが、セクション内に「ポラロイド風」「不揃いの角丸」など、1枚ずつ写真を置くブロックがあると「手作り感」が伝わる |
| **装飾** | 細い線・点・手書き風のアンダーラインなど、ごく小さなデコレーション要素（CSS または SVG）で温かさを足す |
| **プライマリCTA** | 「メニューを見る」「予約する」など、角丸の塗りボタン（茶系やオレンジ）。ホバーで「ぷるん」と scale(1.02) 程度 |
| **ナビ** | ロゴ＋「こだわり」「メニュー」「アクセス」など、やわらかい文言。ヘッダーは背景に溶けてもよい |
| **フッター** | 茶系の細い線で区切り、ロゴまたは店名・住所・営業時間。フォントは本文と同じで、少し小さく |
| **spring 感** | 仕様の「ぷるん」は hover のみ。スクロールでセクションが入ってくるときに、軽い bounce や transition を入れると B-1 らしさが増す |

---

## B-2：Pop & Friendly（楽しさと躍動）

**現状あるもの**: 8px 影、border-4 black、黄・ピンク、ボタン :active scale

| 観点 | 不足している要素（言語化） |
|------|---------------------------|
| **タイポ** | 見出しに「 display 系」のフォント（丸ゴシックやポップ体）を1つ足すと、楽しさがはっきりする |
| **CTA** | 主役は「ピンクの塗り＋黒枠＋影」、サブは「黄の塗り＋黒枠」など、2段階のボタンスタイル。ホバーで少し浮く（translateY(-2px)＋影を強く） |
| **アイコン・イラスト** | 箇条書きやメニューに、シンプルなアイコン（○△□や線アイコン）を置くか、1つ「ゆるいイラスト」を入れると差がつく |
| **フッター** | 同じ「border-4 + 8px 影」の言語で、フッターエリアを囲むか、SNS アイコンを同じスタイルで並べる |
| **セクション** | 吹き出し（voice bubble）風の引用や、バッジ（「人気」「NEW」）を置くブロックがあるとポップさが増す |
| **インタラクション** | カードやボタンのホバーで「影が 8px→12px に」など、動きを少し大げさにすると B-2 らしい |

---

## B-3：High Energy（力強さと勢い）

**現状あるもの**: 上部 marquee、skew セクション、font-black

| 観点 | 不足している要素（言語化） |
|------|---------------------------|
| **ヒーロー** | 中央テキストのみ。全面に「トレーニング・スポーツ」系の写真を敷き、その上にキャッチ＋CTA を載せると「エネルギー」が伝わる |
| **プライマリCTA** | 「無料体験」「資料請求」など、太枠・太字の塗りボタン。ホバーで scale(1.02) や色を少し濃く |
| **数字** | 「3ヶ月」「○○%」など、大きな数字を見せるブロック。marquee と同様に、フォントは black で |
| **ナビ** | ロゴ＋「プログラム」「料金」「アクセス」＋CTA。marquee の下に固定ヘッダー。背景は #0f0f0f のまま |
| **フッター** | 黒背景のまま、skew を活かして斜めに区切るか、直線で区切ってプログラム名・住所・電話を並べる |
| **動き** | セクションが入ってくるときに、skew や x 方向のスライドを軽く入れると「勢い」が揃う |
| **アクセント** | 1色（黄・オレンジ・赤など）を CTA や数字にだけ使い、メリハリをつける |

## 審査員視点：投票できるか・何が足りないか

**結論（審査員として）**  
現状のままでは「投票しづらい」。静的構成・タイポ・余白・CTA は整っていても、**「動き」がほとんどない**ため、コンテストで差がつく「体験の質」が弱い。特に **1〜3（A-1 / A-2 / A-3）は高級感・映画感・信頼感を“見せる”ために、動的な要素がほしい**。4〜6（B-1 / B-2 / B-3）は世界観によっては静的なままでも成立しうるが、1〜3では動きがないと「どこかで見たテンプレ」感が残り、受賞レベルに届きにくい。

---

## 足りない要素：動的なもの（1〜3 で特に重要）

コンテスト上位サイトでは、次のような**動的・時間軸のある表現**がよく使われる。現状はほぼ静止画＋スクロールフェードのみで、以下が不足している。

| 観点 | 現状 | 不足している動き（言語化） |
|------|------|---------------------------|
| **写真の切り替え** | ヒーロー・セクションとも1枚固定 | **スライド／カルーセル／フェード**で複数枚を切り替える。高級ホテル・バー・会社紹介では「何枚か見せる」ことで世界観が伝わる。 |
| **ズーム・パン（Ken Burns 等）** | A-3 で Ken Burns の記述はあるが未実装 or 弱い | ヒーローやメインビジュアルで **ゆっくりズームイン or パン** すると、奥行きと没入感が出る。1〜3 では効果大。 |
| **ヒーローの動画 or スライド** | 静止画1枚 | 背景が **動画ループ** または **複数画像のスライド** だと「入り口」のインパクトが増す。A-1 / A-2 で欲しい。 |
| **スクロール連動の軽い動き** | A-1 のフェードイン程度 | セクションが視界に入るときの **わずかな y 移動・opacity・scale**。A-2 の「映画っぽさ」、A-3 の「信頼感」の演出に有効。 |
| **ホバーでの画像の動き** | ほぼなし | カードやギャラリーで **hover 時に軽いズーム（scale）や明るさ変化** があると、1〜3 の「質感」が伝わる。 |
| **テキストの軽いアニメ** | なし | 見出しの **1行ずつ or 単語ずつのフェードイン** など、控えめなタイポアニメが A-1 などで効く。 |

---

## 1〜3 と 4〜6 の扱い方針

- **1〜3（A-1 Minimal Luxury / A-2 Dark Edge / A-3 Corporate Trust）**  
  **動的なものが足りない**。審査では「写真が切り替わる」「ズームインする」「スクロールで少し動く」といった**時間軸のある演出**があると、「作ってある感」と「体験の質」で差がつく。この3つには少なくとも以下を検討したい。
  - ヒーロー：複数画像のスライド or 1枚でも Ken Burns（ズーム／パン）
  - セクション画像：カルーセル or ホバーで軽いズーム
  - スクロール：セクション進入時のフェード＋軽い y 移動 or scale

- **4〜6（B-1 Warm Organic / B-2 Pop & Friendly / B-3 High Energy）**  
  **静的なものでもよいかもしれない**。B-1 は「温かさ・手作り感」、B-2 は「ポップ・遊び心」、B-3 は「勢い」が主で、すでに marquee や hover scale など少し動きがある。無理にスライドや Ken Burns を足すより、**色・形・タイポ・余白**をきちんと揃える方が効く可能性がある。動きを足すなら「スクロールで軽く出てくる」「ホバーでぷるん」程度で十分な場合がある。

## デザイン評価：賞＋「10万円安い」と思わせる完成度になっているか

### 結論

**現状のままでは「コンテストで賞を取れる範囲かつ10万円安いと思わせる完成度」には届いていない**と判断するのが妥当です。動き・構成・CTA・フッターはそろってきている一方で、**「余白の取り方」がまだチープ**で、全体の「値段感」を下げています。ここを設計し直すと、賞レベルの見えと「安くないのに安いと思わせる」ギャップがつきやすくなります。

---

### なぜ余白がチープに見えるか（要因の言語化）

| 要因 | 現状 | チープに見える理由 |
|------|------|-------------------|
| **スケールがバラバラ** | 1rem / 2rem / 4rem / 6rem / 8rem など、統一した「倍数」がない | 余白にリズムがなく、なんとなく詰めた・空けただけに見える |
| **セクション間が均等** | 多くのセクションが同じ padding（例: 8rem 0）で並ぶ | 呼吸や強弱がなく、単調で安っぽい |
| **コンテナ幅と左右余白の関係が弱い** | 2rem / 6rem など、画面幅に応じた「比率」が曖昧 | 大画面で妙に狭く見えたり、逆にだらしなく広く見えたりする |
| **見出しと本文の間が一律** | h2 の margin 0 0 1rem など、要素間が同じ | タイポの階層（見出し／リード／本文）が余白で伝わらない |
| **「意図的な不均等」がない** | 左右対称・上下均等に寄せがち | 高級感のあるサイトは、あえて余白を「偏らせて」視線を誘導している |
| **フッター・CTA周りが詰まる** | 2rem や 4rem で収めている | 締めの余白が足りず、安いテンプレ感が出る |

これらが重なると、「ちゃんと作ってはいるが、余白の設計が甘い＝安い」という印象になります。

---

### 何をどうすれば「賞＋10万安いと思わせる」完成度になるか

#### 1. 余白の設計を「スケール」で決める（最優先）

- **8px ベースのスケールを採用する**  
  例: `8, 16, 24, 32, 48, 64, 96, 128, 160 (px or 0.5rem=8px 換算)`  
  これ以外の値は極力使わない。`padding: 2.5rem` のような「中途半端」をやめる。
- **役割ごとにスケールを割り当てる**  
  - 要素内のすき間（見出し–本文など）: 16〜24  
  - ブロック間: 32〜48  
  - セクション間: 48〜96（テンプレートによって 64 / 80 / 96 など強弱をつける）  
  - ヒーロー・フッターの上下: 96〜160  
  こうすると「余白にリズム」ができ、チープさが減る。
- **テンプレートごとに「きれいな数字」を1つ決める**  
  例: A-1 は「セクション padding 80px / 120px」、A-2 は「64px / 96px」など、**テンプレート単位で一貫した2〜3段階**に絞る。

#### 2. セクションごとに強弱をつける（均等にしない）

- **すべてのセクションを同じ padding にしない**  
  トップに近いヒーロー直後はやや広く（例: 96）、中盤は標準（64）、フッター手前はやや広く（80）など、**流れで変える**。
- **「息をつくセクション」を意図的につくる**  
  引用ブロックの前後、ギャラリーの前後など、1か所だけ余白を 1.5 倍にするなど、**意図的な不均等**を入れる。

#### 3. コンテナと左右余白の関係をはっきりさせる

- **max-width と padding の関係をルール化する**  
  例: 「コンテナ max-width 1280px のとき、左右 padding は 48px 以上／96px 以下」など。  
  大画面では `calc((100vw - 1280px) / 2 + 48px)` のように、**はみ出し分の半分＋最小余白**で左右を決めると「余白の意図」が伝わる。
- **テンプレートごとに「ゆとり度」を決める**  
  A-1 はやや広め（96px）、B-2 はやや詰め（32px）など、世界観で左右余白を変える。

#### 4. タイポと余白のセットで「階層」を出す

- **見出しの下の余白を、本文の行間の 1〜1.5 倍にする**  
  例: 行間 1.75 なら、h2 の margin-bottom は 1.5em や 2em など、**文字サイズ・行間と比例した値**にする。
- **リード文と本文の間を「やや広く」**  
  最初の1ブロックだけ margin-bottom を 1.5 倍にするなど、**読む順序が余白で分かる**ようにする。

#### 5. フッター・CTA を「締め」の余白で価格感を出す

- **フッターの上方向の余白を「セクションより多め」にする**  
  例: セクションが 64 なら、フッター前は 96 や 128。  
  「最後まで丁寧に余白を取っている」印象で、安さを払しょくする。
- **CTA ブロックの前後も同様**  
  ヒーロー直後の CTA は、上 48・下 64 など、**下をやや広く**取ると「次のセクションへの導線」がはっきりする。

---

### 実装に落とすときの進め方（提案）

1. **共通の余白スケールを定める**  
   CSS 変数または SCSS で `--space-1: 8px; --space-2: 16px; … --space-8: 64px; --space-12: 96px;` のように定義し、**margin / padding はこのスケールのみ使う**ルールにする。
2. **テンプレートごとに「セクション余白」「コンテナ左右」「フッター上」をスケールで再指定する**  
   現状の 8rem などは、上記スケールにマッピングし直す（例: 8rem → 128px → --space-16）。
3. **セクションごとに強弱を付ける**  
   奇数番目は --space-12、偶数番目は --space-8、など、HTML の順序に応じてクラスや CSS で振り分ける。
4. **見出し・本文・リードの margin を「タイポ階層」に合わせて見直す**  
   一律 1rem をやめ、em またはスケール値で差をつける。

このように**余白を「設計」してから数値化**すると、チープさが減り、「コンテストで賞を取れる範囲かつ10万円安いと思わせる」完成度に近づけます。

## まとめ：承認してほしいポイント

1. **全テンプレート共通**  
   ヘッダー（ナビ＋CTA）、プライマリCTA、フッター構成、セクションのバリエーション、ホバー／フォーカス、スキップリンクを「方針として足す」ことにしてよいか。

2. **テンプレート別**  
   上記 A-1〜B-3 の表で挙げた「不足している要素」を、そのまま or 優先度を付けて「実装対象」として承認してよいか。

3. **実装順**  
   共通（ヘッダー・フッター・CTA の枠組み）を先にし、そのあと A-1 → A-2 → … とテンプレート単位で進める形でよいか。  
   あるいは「全テンプレートでナビを先に」など、観点ごとの順序を指定するか。

4. **動的要素（1〜3 重点）**  
   写真の切り替え（スライド／カルーセル）、ズームイン・Ken Burns、スクロール連動の軽い動き、ホバーでの画像拡大などを **1〜3 で優先して足す** 方針でよいか。4〜6 は静的なまま or 軽い動きのみでよいか。

承認いただけた範囲から、「実装」フェーズに進みます。
