# テンプレ4（Warm Organic・カフェ）ショーケースの作り方

**テンプレ1〜3は触らず、テンプレ4の素材だけ**を実運用と同じ操作で作り、完成データをリポジトリに残します。

## 流れの概要

1. アプリで **実際に編集**する（画面と同じフォーム）。
2. 完成したら **JSON をダウンロード**し、プロジェクト内の所定ファイルに **上書き**する。
3. 次回以降、⓪デザインの「Warm Organic」プレビューも **同じ内容**になります。

データの正本は **`src/data/warm-organic-showcase.json`** です（`warmOrganicCafePreset.ts` がこれを読み込みます）。

---

## 手順

### 1. 開発サーバーを起動

```bash
npm run dev
```

### 2. 「③ PDFから作成」タブを開く

### 3. 「Warm Organic のひな形から編集へ」を押す

- テンプレ4用のひな形が読み込まれ、**ステップ3（編集・プレビュー）** に進みます。
- PDFは使いません（カフェ・テンプレ4専用のショートカットです）。

### 4. 左カラムで内容を編集

- サイト名・見出し・各セクションの本文・画像URL など、**実店用にすべて差し替え**。
- **SEO** もタブ下で編集（タイトル・説明・OG画像URL など）。

**ヒーロー3枚**（カルーセル）は、現状エディタに項目がない場合があります。そのときは **手順6のあと**、ダウンロードした JSON を開き、`content.heroSlides` の URL を3本書き換えてから保存し直してください。

### 5. 右のプレビューで見た目を確認

- テンプレが **Warm Organic** になっていることを確認。

### 6. テンプレとして保存

1. 右下 **「テンプレ4用JSONをダウンロード（上書き用）」** をクリック。
2. 保存された `warm-organic-showcase.json` を、次のパスに **上書き**します。  
   **`webpage/src/data/warm-organic-showcase.json`**
3. 開発サーバーを **一度止めて再起動**（または HMR で反映されればそのままで可）。
4. **⓪ デザイン** タブ → **Warm Organic** のプレビューを開き、内容が更新されているか確認。

（任意）**HTMLをダウンロード** で単体HTMLも取っておけます。

---

## PDFから作る場合（別ルート）

テンプレ4以外や PDF 起点のときは、従来どおり **ステップ1でPDF** → ステップ2で **Warm Organic を選択** → ステップ3で編集します。  
テンプレ4ショーケース用JSONを残す手順は **手順6と同じ**（Warm Organic のときだけ JSON ダウンロードボタンが出ます）。

---

## 補足

- **問い合わせフォーム・SNS・QR** がプレビューに出ない場合：⓪デザインの完成例は `DesignCheckPanel` 側でオプション付き生成しています。③のプレビューはオプションなしのことがあります。HTML出力に含めたい場合は `buildHtml` の `genOptions` を渡す拡張が別途必要です。
- テンプレ1〜3のデモデータは **`warm-organic-showcase.json` とは無関係**です。カフェ用はこの JSON だけ差し替えれば足ります。
